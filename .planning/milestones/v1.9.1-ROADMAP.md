## Milestone Summary

**Key Decisions:**

- Test project methodology: progressive validation → fix → deeper validation → fix (SING → FIX1 → FULL → FIX2 interleaved pattern)
- Calculator test projects must have 3+ business flows sharing core computation code points
- Enhanced collector with sync.Mutex for thread safety, flow_id routing to per-flow files
- Module name gojs-calculator ensures output directory collision avoidance
- Frontend probes in event handlers only (not useEffect) — prevents React strict mode double-invocation
- codepoint.ts copied verbatim from frontend.md template for dual-mode compatibility

**Issues Resolved:**

- Scan 输出非确定性问题（修正程序处理）
- Plan 输出缺少上下文
- Implement 混合模式问题
- 堆栈密度不足（20-60% 目标）
- SPA fallback 路由修复
- Batch response unwrap 修复
- Windows 进程管理（orphaned server processes）
- Collector thread safety (sync.Mutex)

**Issues Deferred:**

- Phase 36 VERIFICATION.md human_needed（跨语言联动已通过测试验证，但正式验证报告需人工确认）

**Technical Debt Incurred:**

- 7 个旧 debug session 未关闭（均为历史问题，不影响当前功能）
- `-race` flag 在 Windows CGO 环境不可用
