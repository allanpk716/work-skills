**In `internal/calculator/calculator.go` — shared path probes (hybrid approach):**

The shared `Evaluate()` function should extract flow_id from context and insert probes at key pipeline stages. This is the "hybrid approach" — entry probes at flow boundaries differentiate flows, AND shared path probes capture the computation pipeline:

```go
// flowIDKey is the context key for flow_id propagation.
type flowIDKey struct{}

// GetFlowID extracts flow_id from context.
func GetFlowID(ctx context.Context) string {
    if v, ok := ctx.Value(flowIDKey{}).(string); ok {
        return v
    }
    return "unknown"
}

func Evaluate(ctx context.Context, expr string) (string, error) {
    flowID := GetFlowID(ctx)

    // Shared probe: Parse entry
    codepoint.PointWithMeta("cp-calc-parse", map[string]any{
        "point_id": "cp-calc-parse",
        "flow_id":  flowID,
        "expr":     expr,
    })

    tokens, err := Parse(expr)
    if err != nil {
        return "", err
    }

    // Shared probe: Validate entry
    codepoint.PointWithMeta("cp-calc-validate", map[string]any{
        "point_id": "cp-calc-validate",
        "flow_id":  flowID,
    })

    if err := Validate(tokens); err != nil {
        return "", err
    }

    // Shared probe: Compute entry (state-change)
    codepoint.PointWithMeta("cp-calc-compute", map[string]any{
        "point_id": "cp-calc-compute",
        "flow_id":  flowID,
        "expr":     expr,
    })

    result, err := Compute(tokens)
    if err != nil {
        return "", err
    }

    // Shared probe: Format entry
    codepoint.PointWithMeta("cp-calc-format", map[string]any{
        "point_id": "cp-calc-format",
        "flow_id":  flowID,
        "result":   result,
    })

    return Format(result), nil
}
```

**Key design decision (hybrid approach):**
- **Entry probes** (in server.go, processor.go): differentiate flows at the macro level — stack traces show different handler/processor frames
- **Shared path probes** (in calculator.go Evaluate): capture the computation pipeline — stack traces show different frames because call chains differ (HandleCalculate vs HandleHistoryGet vs ProcessExpressions all call Evaluate, but the stack above Evaluate differs)
- **flow_id in context**: propagates automatically from entry to shared path without modifying every function signature

**Note:** The exact probe positions and IDs are determined by the implement skill based on plan output. The above examples show the EXPECTED pattern. If implement chooses different positions, that is acceptable as long as:
1. Each flow has at least 1 entry probe
2. Shared path has at least 1 probe
3. flow_id is present in all probe meta
4. Probes use PointWithMeta V2 template
