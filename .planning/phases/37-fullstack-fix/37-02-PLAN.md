Replace with the proven implementation from pyts-calculator that writes meta, including inline comments explaining the meta handling:

```python
def receive(entry: dict[str, Any]) -> bool:
    """Write a frontend code point entry. Thread-safe. Returns False if disabled."""
    if not _ts_enabled or _ts_out is None:
        return False
    name = entry.get("name", "unknown")
    stack = entry.get("stack", "")
    meta = entry.get("meta")  # Extract meta dict containing flow_id for cross-language correlation
    with _ts_lock:
        if meta:
            import json as _json
            # Write meta as JSON line so flow_id is available for matching across language boundaries
            _ts_out.write(f"[CODEPOINT] {name}\n{stack}\nmeta: {_json.dumps(meta)}\n")
        else:
            # No meta -- use legacy plain-text format for backward compatibility
            _ts_out.write(f"[CODEPOINT] {name}\n{stack}\n")
        _ts_out.flush()
    return True
```

The key change is: extract `meta` from the entry dict, and if it exists, write it as a JSON line after the stack. This ensures flow_id (inside meta) is available for cross-language correlation. The inline comments explain WHY meta is written as JSON and WHY the else branch preserves backward compatibility.
