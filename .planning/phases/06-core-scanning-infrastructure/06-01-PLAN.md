   使用 dataclass 定义 DetectionRule 基类:
   ```python
   from dataclasses import dataclass
   from typing import Pattern, List, Optional
   import re

   @dataclass
   class DetectionRule:
       """检测规则基类"""
       rule_id: str                    # 规则 ID (如 SENS-01)
       description: str                # 规则描述(中文)
       pattern: Pattern[str]           # 正则表达式模式
       tags: List[str]                 # 分类标签
       entropy_threshold: Optional[float] = None  # 熵值阈值(Phase 7 使用)

       @classmethod
       def create(cls, rule_id: str, description: str,
                  pattern: str, tags: List[str]) -> 'DetectionRule':
           """工厂方法,编译正则表达式"""
           return cls(
               rule_id=rule_id,
               description=description,
               pattern=re.compile(pattern, re.IGNORECASE | re.MULTILINE),
               tags=tags
           )
   ```

4. **实现具体检测规则 (参考 RESEARCH.md 的代码示例):**

   - **SENS-01: AWS 凭证检测**
     * AWS_ACCESS_KEY_RULE: `r'\b((?:A3T[A-Z0-9]|AKIA|ASIA|ABIA|ACCA)[A-Z0-9]{16})\b'`
     * AWS_SECRET_KEY_RULE: `r'(?i)aws(.{0,20})?(?-i)[\'\"]?[0-9a-zA-Z\/+]{40}[\'\"]?'`
     * AWS_SESSION_TOKEN_RULE: `r'(?i)(aws_session_token|awssessiontoken|session_token)(?:.{0,20})?[\'\"]?[a-zA-Z0-9/+=]{16,}[\'\"]?'`

   - **SENS-02: Git 服务 token 检测**
     * GITHUB_TOKEN_RULE: `r'\b(ghp_[a-zA-Z0-9]{36}|github_pat_[a-zA-Z0-9]{22}_[a-zA-Z0-9]{59})\b'`
     * GITLAB_TOKEN_RULE: `r'\b(glpat-[a-zA-Z0-9\-_]{20})\b'`
     * BITBUCKET_TOKEN_RULE: `r'(?i)bitbucket(.{0,20})?[\'\"]?[a-zA-Z0-9]{32}[\'\"]?'`

   - **SENS-03: 通用 API 密钥检测**
     * GENERIC_API_KEY_RULE: GitLeaks generic 规则

     r'(?i)[\w.-]{0,50}?(?:access|auth|(?-i:[Aa]pi|API)|credential|creds|key|passw(?:or)?d|secret|token)'
     r'(?:[ \t\w.-]{0,20})[\s\'"]{0,3}'
     r'(?:=||:{1,3}=|\|\||:|=|\?=|,)'
     r'[\x60\'"\s=]{0,5}'
     r'([\w.=-]{10,150}|[a-z0-9][a-z0-9+/]{11,}={0,3})'

3. **创建 git_ops.py,实现暂存区文件获取:**

   ```python
   import subprocess
   from pathlib import Path
   from typing import List, Tuple, Optional

   def get_staged_files(repo_root: Path) -> List[Tuple[Path, str]]:
       """
       获取暂存区文件列表和内容

       Args:
           repo_root: Git 仓库根目录

       Returns:
           List of (file_path, content) tuples

       Raises:
           subprocess.CalledProcessError: Git 命令执行失败
       """
       # 获取暂存区文件列表
       result = subprocess.run(
           ['git', 'diff', '--cached', '--name-only'],
           cwd=repo_root,
           capture_output=True,
           text=True,
           check=True
       )

       file_paths = [repo_root / p for p in result.stdout.strip().split('\n') if p]

       # 获取每个文件的内容
       files_with_content = []
       for file_path in file_paths:
           try:
               content = file_path.read_text(encoding='utf-8', errors='ignore')
               files_with_content.append((file_path, content))
           except Exception:
               # 跳过无法读取的文件(二进制文件等)
               continue

       return files_with_content

   def is_binary_file(file_path: Path) -> bool:
       """
       检测文件是否为二进制文件

       Args:
           file_path: 文件路径

       Returns:
           True 如果是二进制文件, False 如果是文本文件
       """
       binary_extensions = {
           '.exe', '.dll', '.so', '.dylib', '.png', '.jpg', '.jpeg', '.gif',
           '.pdf', '.zip', '.tar', '.gz', '.class', '.jar', '.war'
       }

       if file_path.suffix.lower() in binary_extensions:
           return True

       # 检查文件内容的前 8192 字节
       try:
           with open(file_path, 'rb') as f:
               chunk = f.read(8192)
               return b'\x00' in chunk  # NULL 字节表示二进制文件
       except Exception:
           return True  # 无法读取,保守处理为二进制文件
   ```

**实现注意事项:**
- 使用 subprocess.run 调用 git 命令,不要使用 GitPython (避免外部依赖)
- 使用 pathlib.Path 处理路径,确保 Windows 兼容性
- 读取文件时使用 errors='ignore' 忽略编码错误
- 实现二进制文件检测 (扩展名白名单 + NULL 字节检测)
- 只扫描暂存区文件 (git diff --cached),不要扫描整个仓库
