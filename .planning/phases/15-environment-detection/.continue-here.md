<decisions_made>

### Phase 15 关键决策

1. **使用 execa 而不是 child_process**
   - 原因: 更好的 Windows 支持,Promise API,自动处理路径问题
   - 版本选择: 降级到 5.1.1 以兼容 CommonJS 架构

2. **使用 winreg 而不是 REG 命令**
   - 原因: 类型安全的 API,避免命令输出解析问题
   - 支持 Wow6432Node 回退(32 位应用在 64 位 Windows)

3. **SSH 工具检测接受任一工具**
   - 原因: TortoiseGit 或 PuTTY 都可以用于 Git SSH 认证
   - 提高灵活性,不强制要求特定工具

4. **并行运行所有检测器**
   - 原因: 使用 Promise.all 提高检测速度
   - 所有检测相互独立,无依赖关系

5. **i18n 支持参数替换**
   - 原因: 翻译字符串需要动态值(如 {passed}/{total})
   - 增强 t() 函数支持参数对象

6. **检测器返回统一结构**
   - 原因: 便于集成层统一处理和格式化输出
   - 结构: { name, installed, version, meetsMinimum, guidance, ... }

</decisions_made>
