    ```python
    # 2. Notification Flags (per D-01, D-02)
    print("\n[2] Notification Flags")
    flags = check_notification_flags()

    # Pushover
    if flags['pushover_disabled']:
        path = flags['pushover_path'] or flags['global_pushover_path']
        source = "project-level" if flags['pushover_path'] else "global"
        print(f"  Pushover: DISABLED - Found at {path} ({source})")
    else:
        print("  Pushover: Enabled (no .no-pushover found)")

    # Windows
    if flags['windows_disabled']:
        path = flags['windows_path'] or flags['global_windows_path']
        source = "project-level" if flags['windows_path'] else "global"
        print(f"  Windows: DISABLED - Found at {path} ({source})")
    else:
        print("  Windows: Enabled (no .no-windows found)")
    ```

    Key implementation details:
    - `check_notification_flags` is already imported at line 21: `from flags import check_notification_flags`
    - The `path = flags['pushover_path'] or flags['global_pushover_path']` pattern works because project-level is checked first; if project path exists it's used, otherwise global path is used
    - The `source = "project-level" if flags['pushover_path'] else "global"` correctly labels the source because `pushover_path` is only set when found at project-level (upward traversal), while `global_pushover_path` is only set when found at ~/.claude/
    - When neither path exists (shouldn't happen if disabled=True, but defensive), the output still makes sense
