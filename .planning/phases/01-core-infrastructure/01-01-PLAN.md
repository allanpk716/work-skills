<task type="auto">
  <name>Task 2: 实现通知脚本主文件</name>
  <files>.claude/hooks/notify.py</files>
  <action>
创建通知脚本主文件,实现完整的 notification 流程:生成摘要 → 并行发送通知 → 严格超时控制。

核心功能模块:

1. **项目名称获取 (CORE-03)**
   - 使用 `os.path.basename(os.getcwd())` 获取当前工作目录名称
   - UTF-8 编码,支持中文字符

2. **Claude CLI 摘要生成 (SUMM-01, SUMM-02, SUMM-03, SUMM-04)**
   - 使用 `subprocess.run()` 调用 `claude --print`
   - 提示词: "Summarize the completed task in one short sentence (max 200 chars)."
   - timeout=2 (2 秒超时)
   - 失败降级消息: f"[{project_name}] Task completed"
   - 截断到 200 字符: `summary[:200]`

3. **Pushover 通知发送 (PUSH-01, PUSH-02, PUSH-03, PUSH-04, PUSH-05)**
   - 使用 `requests.post()` 调用 `https://api.pushover.net/1/messages.json`
   - 参数: token, user, title(项目名), message(摘要), priority=0
   - timeout=2 (2 秒超时)
   - 环境变量: `os.environ.get('PUSHOVER_TOKEN')`, `os.environ.get('PUSHOVER_USER')`
   - 错误处理: 缺失环境变量时返回 False,记录日志但不抛出异常

4. **Windows Toast 通知发送 (WIND-01, WIND-02, WIND-03, WIND-04, WIND-05)**
   - 使用 `subprocess.run()` 调用 PowerShell
   - PowerShell 脚本使用 Windows.UI.Notifications API
   - Toast XML 模板: ToastText02 (标题 + 内容)
   - timeout=1 (1 秒超时)
   - 失败降级: 返回 False,不影响其他通道

5. **并行通知发送 (PARA-01, PARA-02, PARA-03)**
   - 使用 `concurrent.futures.ThreadPoolExecutor(max_workers=2)`
   - 并行提交 Pushover 和 Windows 通知任务
   - 使用 `as_completed(futures, timeout=4)` 等待结果
   - 总体超时 4 秒,确保在 5 秒限制内
   - 每个通知通道独立 try/except,失败不影响另一个

6. **错误处理和日志 (CORE-04)**
   - 日志文件路径: `%APPDATA%\claude-notify\logs\claude-notify-{YYYYMMDD}-{PID}.log`
   - 使用 Python logging 模块
   - 记录关键步骤和错误信息
   - PID 隔离确保多实例安全

实现细节:
- 不使用 `shell=True`,直接传递命令列表
- 所有 subprocess 调用使用 `capture_output=True, text=True`
- 环境变量缺失时静默失败,记录日志但不崩溃
- UTF-8 编码处理所有字符串

关键约束:
- 严格控制在 5 秒内完成 (CORE-02)
- 不实现重试逻辑,快速失败策略
- 不使用共享状态,确保并发安全 (CORE-04)
