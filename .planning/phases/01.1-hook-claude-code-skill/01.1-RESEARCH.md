## Summary

Phase 01.1 需要将 Phase 1 实现的 Hook 通知脚本转换为标准 Claude Code 插件格式,通过 GitHub 市场分发。研究涵盖了 Claude Code 插件架构、hooks.json 配置格式、marketplace.json 市场配置、以及从项目级 Hook 迁移到插件级 Hook 的关键差异。

**Primary recommendation:** 使用 skills 目录结构封装通知功能,通过 `hooks/hooks.json` 定义 Stop 事件 Hook,利用 `${CLAUDE_PLUGIN_ROOT}` 实现路径可移植性,通过 GitHub 仓库作为插件市场分发。

### Supporting
| Component | Version | Purpose | When to Use |
|-----------|---------|---------|-------------|

| Python 3.x | 3.8+ | Hook 脚本运行时 | 通知脚本执行 |
| requests library | stdlib | HTTP 请求 | Pushover API 调用 |

### Key Differences: Project vs Plugin Hooks

| Aspect | Project Hook | Plugin Hook |
|--------|-------------|-------------|
| Config Location | `.claude/settings.json` | `hooks/hooks.json` |
| Config Format | Direct format (no wrapper) | Wrapper format with `hooks` key |
| Path Variables | `$CLAUDE_PROJECT_DIR` | `${CLAUDE_PLUGIN_ROOT}` |
| Distribution | Project-specific | Market-distributed |

**Project Hook Format (Phase 1 当前实现):**
```json
{
  "hooks": {
    "Stop": [...]
  }
}
```

**Plugin Hook Format (Phase 01.1 目标):**
```json
{
  "description": "Notification hooks for task completion",
  "hooks": {
    "Stop": [...]
  }
}
```

## Architecture Patterns

### Recommended Plugin Structure

```
claude-notify-plugin/
├── .claude-plugin/
│   └── marketplace.json      # Market definition (NOT plugin.json for marketplaces)
├── skills/
│   └── claude-notify/
│       └── SKILL.md          # Skill documentation
├── hooks/
│   ├── hooks.json            # Hook configuration
│   └── scripts/
│       └── notify.py         # Main notification script
├── scripts/
│   └── verify-installation.py  # Installation verification
├── tests/
│   └── test_notify.py        # Test script
└── README.md                 # Installation and usage guide
```

### Pattern 1: Plugin Hook Configuration

**What:** hooks.json 定义 Stop 事件触发的通知脚本
**When to use:** 插件级 Hook 配置

```json
{
  "description": "Sends notifications when Claude Code tasks complete",
  "hooks": {
    "Stop": [
      {
        "matcher": "*",
        "hooks": [
          {
            "type": "command",
            "command": "python \"${CLAUDE_PLUGIN_ROOT}/hooks/scripts/notify.py\"",
            "async": true,
            "timeout": 10
          }
        ]
      }
    ]
  }
}
```

**Source:** Claude Code Plugin Dev documentation

### Pattern 2: Marketplace Configuration

**What:** marketplace.json 定义市场元数据和插件列表
**When to use:** GitHub 仓库作为插件市场

```json
{
  "name": "claude-notify-marketplace",
  "owner": {
    "name": "Developer Name",
    "email": "email@example.com"
  },
  "metadata": {
    "description": "Claude Code notification plugins",
    "version": "1.0.0"
  },
  "plugins": [
    {
      "name": "claude-notify",
      "description": "Task completion notifications via Pushover and Windows Toast",
      "source": "./",
      "strict": false,
      "skills": [
        "./skills/claude-notify"
      ]
    }
  ]
}
```

**Source:** work-skills marketplace example

### Pattern 3: Path Portability with ${CLAUDE_PLUGIN_ROOT}

**What:** 使用环境变量确保路径可移植性
**When to use:** 所有 Hook 命令中的路径引用

```json
{
  "type": "command",
  "command": "python \"${CLAUDE_PLUGIN_ROOT}/hooks/scripts/notify.py\""
}
```

**Key points:**
- 插件可能安装在不同位置(用户目录、市场缓存等)
- 绝对路径会在不同用户环境中失效
- `${CLAUDE_PLUGIN_ROOT}` 在运行时解析为实际安装路径

### Anti-Patterns to Avoid

- **Hardcoded paths:** 使用 `C:\Users\...` 或 `/home/user/...` 会导致在其他环境失效
- **Project-level config in plugin:** 在 `hooks/hooks.json` 中使用 `$CLAUDE_PROJECT_DIR`
- **Missing hooks wrapper:** 忘记在 `hooks.json` 中使用 `{"hooks": {...}}` 包装格式
- **Plugin.json for marketplace:** 市场使用 `marketplace.json`, 单独插件使用 `plugin.json`

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Hook config format | Custom JSON schema | Standard hooks.json wrapper format | Claude Code 期望特定格式 |
| Path resolution | Custom path logic | ${CLAUDE_PLUGIN_ROOT} | 运行时自动解析 |
| Market metadata | Custom config file | marketplace.json | 市场发现和安装依赖此格式 |
| Skill documentation | Custom doc format | SKILL.md with frontmatter | 自动发现机制需要 |

**Key insight:** Claude Code 插件系统有严格的约定,偏离约定会导致组件无法被发现或加载。

## Common Pitfalls

### Pitfall 1: Wrong hooks.json Format

**What goes wrong:** 使用项目级 Hook 格式而不是插件级格式
**Why it happens:** Phase 1 使用的是 `.claude/settings.json` 中的项目级格式
**How to avoid:**
- 插件 hooks.json 必须包含 `description` 和 `hooks` 包装器
- 项目 settings.json 直接使用事件名作为顶级键

**Wrong (project format):**
```json
{
  "Stop": [...]
}
```

**Correct (plugin format):**
```json
{
  "description": "Notification hooks",
  "hooks": {
    "Stop": [...]
  }
}
```

**Warning signs:** Claude Code --debug 显示 hooks 未加载

### Pitfall 2: Missing CLAUDE_PLUGIN_ROOT

**What goes wrong:** 使用硬编码路径或 $CLAUDE_PROJECT_DIR
**Why it happens:** 从 Phase 1 迁移时直接复制配置
**How to avoid:**
- 在所有 hook command 中使用 `${CLAUDE_PLUGIN_ROOT}`
- 注意大括号语法: `${}` 不是 `$`

**Wrong:**
```json
"command": "python \"$CLAUDE_PROJECT_DIR\\.claude\\hooks\\notify.py\""
```

**Correct:**
```json
"command": "python \"${CLAUDE_PLUGIN_ROOT}/hooks/scripts/notify.py\""
```

### Pitfall 3: marketplace.json vs plugin.json

**What goes wrong:** 在市场仓库根目录使用 plugin.json
**Why it happens:** 混淆了单独插件和市场仓库的结构
**How to avoid:**
- GitHub 市场仓库: 根目录使用 `.claude-plugin/marketplace.json`
- 单独插件: 根目录使用 `.claude-plugin/plugin.json`

### Pitfall 4: Skills Directory Structure

**What goes wrong:** SKILL.md 文件命名或位置错误
**Why it happens:** 技能目录结构有严格要求
**How to avoid:**
- 必须在 `skills/<skill-name>/SKILL.md` 路径
- 不能是 `skills/<skill-name>.md` 或 `skills/<skill-name>/README.md`
