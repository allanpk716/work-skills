<context>
Phase 20 is about enhancing the installer's existing configurator modules (pushover.js, git-user.js) to detect existing configurations and offer smart skip/update choices. The key insight is that `setx` persists env vars to the Windows registry but `process.env` only reflects them after terminal restart — so we need dual-source detection (D-01).

The two plans are independent in terms of files modified but logically sequential: Plan 01 (Pushover) is wave 1, Plan 02 (Git user) is wave 2. Each plan has 2 tasks: code implementation + i18n/tests.

Plans contain detailed code snippets for implementation — the executor should follow them closely. Both configurators use the enquirer library (Confirm + Input prompts) and the t() i18n function.

Key files to modify:
- `installer/src/configurators/pushover.js` — add readRegistryEnvVar, detectPushoverFull, per-item Confirm
- `installer/src/configurators/git-user.js` — add per-item Confirm (4 cases)
- `installer/src/i18n/en.json` and `zh.json` — 6 new keys per configurator
- `installer/tests/configurators/pushover.test.js` and `git-user.test.js` — extend tests
</context>
