# Project Guidelines for Claude Code

## 交流语言
- 使用中文回答问题和进行交流

## 开发环境
- 开发系统是 **Windows**
- 注意使用 Windows 兼容的路径分隔符和命令

## 代码编写规范

### 脚本开发
- 开发、编辑 BAT 脚本时,**不要在脚本中包含中文**
- 修复脚本时,在原有脚本上进行修改,**非必需不要新建脚本**

### 临时文件管理
- 临时测试代码、数据统一放在项目根目录的 `tmp` 文件夹中

## 文档和计划管理

### 计划文件
- 项目的计划文件放在 `docs\plans` 目录中

### Bug 记录
- 项目自测发现的 bug 记录在 `docs\bugs` 目录中

## 测试规范

### 前后端分离项目
- 如果是前后端分离开发的项目,使用浏览器(**dev-browser 技能**)进行测试
- 重点测试前后端的通信和交互操作

### 图片处理
- 如果需要使用截图的 MCP 或 Agent 能力,在提交到识别图片服务之前,**确保图片尺寸小于 1000x1000**

## 插件开发规范

### 开发原则
- **单一加载源**: 插件只从一个位置加载,避免重复
- **生命周期分离**: 开发阶段和发布阶段使用不同的加载方式
- **定期清理**: 插件结构变化时清理缓存

### 目录结构
- ✅ 使用 `plugins/<name>/skills/<skill>/SKILL.md` 定义技能
- ❌ 避免使用 `commands/` 目录(会导致命令重复)
- ❌ 不要同时存在旧的 `skills/` 根目录

### 开发阶段
- 在 `plugins/` 目录中开发
- **不要**添加到 `marketplace.json`
- 使用本地路径加载

### 发布阶段
- 从 `plugins/` 目录移除
- 添加到 `marketplace.json`
- 清理缓存后重启

- [插件开发最佳实践](docs/plugin-development-best-practices.md)
- [快速参考卡片](docs/plugin-quick-reference.md)

## 工作原则
- 优先在现有文件上修改,避免创建不必要的文件
- 保持代码和文档的简洁性
- 遵循项目现有的代码风格和目录结构
