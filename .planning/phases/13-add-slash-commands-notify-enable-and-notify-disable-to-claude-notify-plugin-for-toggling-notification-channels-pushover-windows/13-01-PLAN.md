# Plan 01: Implement Slash Commands

## Objective

实现三个斜杠命令(`/notify-enable`、`/notify-disable`、`/notify-status`),通过操作 `.no-pushover` 和 `.no-windows` 标志文件来控制通知通道的启用/禁用状态及查询当前状态。

## What This Builds

- `/notify-enable <pushover|windows>` - 启用指定通知通道
- `/notify-disable <pushover|windows>` - 禁用指定通知通道
- `/notify-status` - 查询所有通道状态

## Tasks

### Task 1: Create notify-enable.py script

<task>
Create `plugins/claude-notify/scripts/notify-enable.py` to enable notification channels by removing flag files.

**Requirements:**
- Accept single parameter: `pushover` or `windows`
- Validate parameter and show error if invalid or missing
- Check current state (file existence) before operation
- Remove `.no-{channel}` file if it exists
- Provide clear feedback messages in Chinese:
  - Success: "{Channel} 通知已启用"
  - Already enabled: "{Channel} 通知已处于启用状态"
  - Invalid parameter: "错误:无效参数 '{param}'。可用选项:pushover, windows。用法:/notify-enable <pushover|windows>"
  - Missing parameter: "错误:缺少参数。用法:/notify-enable <pushover|windows>"
- Handle file operation errors gracefully
- Use idempotent operations

**Implementation pattern:**
```python
#!/usr/bin/env python3
import sys
from pathlib import Path

VALID_CHANNELS = {'pushover', 'windows'}

def enable_channel(channel: str) -> str:
    """Enable a notification channel by removing flag file."""
    flag_file = Path.cwd() / f".no-{channel}"

    if not flag_file.exists():
        return f"{channel.capitalize()} 通知已处于启用状态"

    try:
        flag_file.unlink()
        return f"{channel.capitalize()} 通知已启用"
    except OSError as e:
        return f"错误:无法删除标志文件 - {e}"

def main():
    if len(sys.argv) != 2:
        print("错误:缺少参数。用法:/notify-enable <pushover|windows>")
        sys.exit(1)

    channel = sys.argv[1].lower()

    if channel not in VALID_CHANNELS:
        print(f"错误:无效参数 '{channel}'。")
        print(f"可用选项:{', '.join(sorted(VALID_CHANNELS))}")
        print(f"用法:/notify-enable <pushover|windows>")
        sys.exit(1)

    result = enable_channel(channel)
    print(result)

if __name__ == "__main__":
    main()
```

**Verification:**
- Script runs without errors
- Validates parameters correctly
- Removes correct flag file
- Handles already-enabled state
- Shows appropriate messages

### Task 3: Create notify-status.py script

<task>
Create `plugins/claude-notify/scripts/notify-status.py` to query notification channel status.

**Requirements:**
- No parameters needed
- Check existence of `.no-pushover` and `.no-windows` files
- Display status of all channels using icons:
  - Enabled: `✓ 已启用`
  - Disabled: `✗ 已禁用`
- Output format (example):
  ```
  Pushover 通知: ✓ 已启用
  Windows 通知: ✗ 已禁用
  ```
- Handle file read errors gracefully

**Implementation pattern:**
```python
#!/usr/bin/env python3
from pathlib import Path

def get_channel_status(channel: str) -> str:
    """Get status of a notification channel."""
    flag_file = Path.cwd() / f".no-{channel}"
    is_enabled = not flag_file.exists()
    status_icon = "✓" if is_enabled else "✗"
    status_text = "已启用" if is_enabled else "已禁用"
    return f"{status_icon} {status_text}"

def main():
    channels = ['pushover', 'windows']

    for channel in channels:
        status = get_channel_status(channel)
        print(f"{channel.capitalize()} 通知: {status}")

if __name__ == "__main__":
    main()
```

**Verification:**
- Script runs without parameters
- Shows correct status for both channels
- Uses correct icons (✓/✗)
- Output format matches specification

## 项目级控制开关 (旧方法)

除了使用斜杠命令,您也可以手动创建标志文件来控制通知:

### 禁用特定项目的 Pushover 通知

在项目根目录创建 `.no-pushover` 文件:

\`\`\`cmd
REM 命令提示符
type nul > .no-pushover

REM 或使用 PowerShell
New-Item -Path .no-pushover -ItemType file
\`\`\`

**效果:**
- 该项目不会发送 Pushover 通知
- Windows Toast 通知继续正常工作
- 适用于个人项目或不需要远程通知的场景

### 禁用特定项目的 Windows 通知

在项目根目录创建 `.no-windows` 文件:

\`\`\`cmd
type nul > .no-windows
\`\`\`

**效果:**
- 该项目不会发送 Windows Toast 通知
- Pushover 通知继续正常工作
- 适用于在共享屏幕演示时避免干扰

### 同时禁用两种通知

创建两个文件:

\`\`\`cmd
type nul > .no-pushover
type nul > .no-windows
\`\`\`

**效果:** 该项目完全禁用所有通知。

**推荐:** 使用斜杠命令 (`/notify-disable`) 更便捷,效果相同。
